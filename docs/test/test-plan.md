## 1. 测试概述

### 1.1 测试范围
- **核心功能模块**：JSON流解析、路径提取、配置管理、依赖树构建
- **UI交互功能**：依赖树展示、文件浏览、跳转定位、只读查看
- **高级功能**：Replace依赖处理、标准库支持、懒加载模式、工作空间支持
- **兼容性**：VSCode API集成、跨平台路径处理、Go版本兼容

### 1.2 测试策略

#### 单元测试
- **目标**：核心业务逻辑和纯函数
- **工具**：Node.js 内置 assert 模块
- **覆盖率目标**：≥ 85%
- **执行频率**：每次代码变更后

#### 集成测试
- **目标**：VSCode API集成、跨模块交互
- **环境**：VSCode Extension Development Host
- **方式**：手动测试验证
- **频率**：版本发布前

#### 回归测试
- **目标**：确保新功能不破坏现有功能
- **重点**：标准库跳转、依赖定位、工作空间支持
- **覆盖**：所有P0功能和关键P1功能

### 1.3 测试环境

**开发环境要求**：
- Node.js 18+
- TypeScript 5.3+
- VSCode 1.75.0+

**测试数据环境**：
- 包含go.mod的Go项目
- 多项目工作空间
- 标准库和第三方依赖混合场景
